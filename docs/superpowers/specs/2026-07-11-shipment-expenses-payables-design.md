# 出运费用、供应商应付与预付款联动设计

日期：2026-07-11  
项目：FUTUREM Enterprise Sourcing ERP  
状态：已与用户确认

## 1. 背景与目标

当前出运单仅包含运输基础信息和商品明细，无法在同一业务单据中记录海运费、仓库费、装卸费、内陆费等出运费用，也无法自动形成对应供应商的应付账款。

本次改造目标：

1. 在出运单中维护固定费用和自定义费用。
2. 每条费用关联一个供应商。
3. 出运单确认或标记已出运后，自动生成供应商应付账款。
4. 出运单确认后允许继续修改费用，并自动同步对应应付。
5. 费用调低导致超付时，将差额转为供应商预付款。
6. 新应付生成时，自动优先抵扣同供应商、同币种的可用预付款。
7. 出运单立方数、毛重、净重由商品明细自动汇总，同时允许人工调整最终值。
8. 金额、立方数、毛重、净重统一保留两位小数。
9. 商品条码允许修改，但必须保持全系统唯一。

## 2. 已确认的业务规则

### 2.1 费用类型

每张出运单默认包含以下固定费用：

- 海运费
- 仓库费
- 装卸费
- 内陆费

同时允许新增其他费用，例如：

- 报关费
- 文件费
- 码头费
- 保险费

规则：

- 每种固定费用只能存在一行。
- 自定义费用按费用名称唯一，同名费用不能重复添加。
- 不允许同一种费用对应多行或多个供应商。

### 2.2 币种

- 每张出运单只能选择一种币种。
- 所有费用、应付、预付款和抵扣记录必须继承出运单币种。
- 不同币种之间不自动抵扣。

### 2.3 供应商

- 费用金额等于 0 时，可以不选择供应商。
- 费用金额大于 0 时，必须选择供应商，否则禁止确认出运单。
- 供应商可以是海运公司、仓库、搬运工、货代、车队或其他服务供应商。

### 2.4 财务生成时点

- 草稿保存时只保存费用，不生成应付。
- 出运单首次变为 `confirmed` 或 `shipped` 时，为每条金额大于 0 的费用分别生成一条供应商应付。
- 重复点击确认、已出运或财务同步时，不得重复生成应付。

### 2.5 确认后修改

- 出运单确认后仍允许修改费用金额、供应商和备注。
- 修改后自动更新对应应付账款。
- 一条出运费用只能对应一条主应付记录。

### 2.6 超付与供应商预付款

当费用金额调低，且新金额小于已付款金额与已抵扣金额之和时：

- 应付金额更新为新费用金额。
- 超出部分单独生成供应商预付款记录。
- 预付款归属原供应商和原币种。
- 预付款可用于后续同供应商、同币种的新应付。

示例：

- 原费用：10,000
- 已付款及抵扣：8,000
- 新费用：7,000
- 结果：生成 1,000 供应商预付款

### 2.7 预付款自动抵扣

新应付生成或金额增加时：

1. 查询同供应商、同币种的可用预付款。
2. 按产生时间优先抵扣。
3. 抵扣金额不超过当前应付未付余额。
4. 保存每一笔抵扣流水。
5. 更新预付款可用余额和应付已抵扣金额。

### 2.8 供应商变更

确认后更换费用供应商时：

- 原供应商没有付款或抵扣：撤销原应付，给新供应商生成新应付。
- 原供应商已有付款或抵扣：不能直接迁移付款。
- 原供应商的超付部分转为其供应商预付款。
- 新供应商重新生成应付，并自动尝试抵扣其同币种预付款。
- 全过程保留操作和财务流水。

## 3. 状态设计

出运单状态固定为：

- `draft`：草稿
- `confirmed`：已确认
- `shipped`：已出运
- `completed`：已完成
- `cancelled`：已取消

规则：

- `draft`：可编辑，不生成应付。
- `confirmed`：生成并同步费用应付。
- `shipped`：生成或继续同步费用应付。
- `completed`：业务结束，但保留费用、应付和付款查询。
- `cancelled`：若存在已付款或已抵扣费用，不允许直接取消，必须先处理付款或转预付款。

## 4. 数据模型设计

### 4.1 Shipment 扩展字段

出运单增加：

- `Currency`
- `CalculatedTotalCbm`
- `FinalTotalCbm`
- `CalculatedGrossWeightKg`
- `FinalGrossWeightKg`
- `CalculatedNetWeightKg`
- `FinalNetWeightKg`
- `ExpenseTotal`
- `FinanceSyncStatus`
- `FinanceSyncMessage`
- `FinanceSyncedAt`

计算字段由出运商品明细自动汇总。最终字段默认等于计算字段，但允许人工调整。

### 4.2 ShipmentExpense 出运费用表

建议新增 `shipment_expenses` 表：

- `Id`
- `ShipmentId`
- `ExpenseCode`
- `ExpenseName`
- `IsCustom`
- `SupplierId`
- `Amount`
- `Currency`
- `FinanceRecordId`
- `FinanceStatus`
- `Remark`
- `SortNo`
- `IsDeleted`
- `CreatedAt`
- `UpdatedAt`

约束：

- `ShipmentId + ExpenseCode` 对固定费用唯一。
- `ShipmentId + normalized ExpenseName` 对自定义费用唯一。
- `Amount > 0` 时 `SupplierId` 必填。
- `Currency` 必须与 Shipment 一致。

### 4.3 SupplierPrepayment 供应商预付款表

建议新增 `supplier_prepayments` 表：

- `Id`
- `No`
- `SupplierId`
- `Currency`
- `OriginalAmount`
- `AvailableAmount`
- `SourceType`
- `SourceId`
- `SourceFinanceRecordId`
- `Status`
- `Remark`
- `CreatedAt`
- `UpdatedAt`

状态：

- `available`
- `partially_used`
- `used`
- `cancelled`

### 4.4 SupplierPrepaymentUsage 预付款使用流水

建议新增 `supplier_prepayment_usages` 表：

- `Id`
- `SupplierPrepaymentId`
- `FinanceRecordId`
- `Amount`
- `UsageType`
- `Remark`
- `CreatedAt`

用途：

- 记录每次预付款抵扣。
- 支持追溯、撤销和财务核对。

### 4.5 FinanceRecord 扩展

为保证每条费用独立关联应付，建议增加：

- `ShipmentExpenseId`
- `PrepaymentAppliedAmount`
- `SourceKey`

约束：

- 每个有效 `ShipmentExpenseId` 最多对应一条主应付。
- 不再仅依赖 `TargetType + TargetId` 判断唯一性。

增加：

- 币种
- 计算立方数
- 最终立方数
- 计算毛重
- 最终毛重
- 计算净重
- 最终净重
- 费用总额
- 财务同步状态

计算值只读。最终值可编辑，统一显示两位小数。

### 5.2 出运费用表格

在商品明细下方增加“出运费用”区域，列为：

- 费用名称
- 供应商
- 金额
- 已付
- 未付
- 预付款抵扣
- 应付状态
- 备注

默认显示四个固定费用。提供“新增其他费用”按钮。

### 5.3 财务状态显示

每条费用显示：

- 未生成
- 待付款

### 5.4 操作按钮

- 确认出运单

- 同步财务
- 关闭

“同步财务”用于异常恢复；正常业务自动同步。

## 6. 体积与重量汇总

系统从出运单商品明细汇总：

- `TotalCbm`
- `TotalGwKg`
- `TotalNwKg`

规则：

- 计算值为所有有效明细行合计。
- 用户点击重新计算或保存商品明细后更新计算值。
- 最终值初次默认等于计算值。
- 用户人工修改最终值后，后续重新计算不得无提示覆盖人工值。
- 页面同时显示计算值和最终值，方便追溯。

## 7. 金额与数值精度

以下字段统一按两位小数保存和显示：

- 费用金额
- 应付金额
- 已付金额
- 预付款金额
- 预付款余额
- 预付款抵扣金额
- 立方数
- 毛重
- 净重
- 商品采购价格及相关金额字段

实现规则：

- 后端使用 `decimal`。
- 数据库使用 `decimal(18,2)`。
- 保存前按两位小数四舍五入。
- 前端输入组件使用 `precision=2`。
- 禁止使用浮点数参与财务计算。

## 8. 商品条码修改

商品编辑接口允许修改条码。

校验规则：

- 去除首尾空格。
- 不允许为空。
- 最大长度 80。
- 全系统唯一。
- 更新时排除当前商品自身。
- 已删除商品条码仍视为占用，避免历史单据追溯混乱。
- 修改主数据条码不回写历史业务单据。

## 9. 服务边界

### 9.1 ShipmentExpenseService

职责：

- 新增、修改、删除出运费用。
- 初始化固定费用。
- 校验费用唯一性。
- 校验供应商必填。
- 汇总费用总额。

### 9.2 ShipmentMeasurementService

职责：

- 汇总商品明细立方数、毛重和净重。
- 更新计算值。
- 保留人工最终值。

### 9.3 ShipmentFinanceSyncService

职责：

- 根据出运费用创建或更新应付。
- 防止重复应付。
- 处理供应商变更。
- 处理金额增加、减少及超付。
- 调用预付款抵扣服务。
- 更新财务同步状态。

### 9.4 SupplierPrepaymentService

职责：

- 生成供应商预付款。
- 查询可用预付款。
- 自动按时间顺序抵扣。
- 保存预付款使用流水。
- 更新余额和状态。

## 10. 事务与幂等

出运单确认、标记已出运或手工同步财务时，必须在单个数据库事务中完成：

1. 校验出运单。
2. 校验费用名称、金额、币种和供应商。
3. 更新出运状态。
4. 为每条费用创建或更新应付。
5. 处理超付和供应商预付款。
6. 自动抵扣可用预付款。
7. 更新费用财务状态。
8. 更新出运单财务同步状态。
9. 全部成功后提交。

任何一步失败：

- 整次操作回滚。
- 出运单保持原状态。
- 不允许出现部分费用已生成应付、部分失败的状态。
- 保存错误原因，允许后续重新同步。

幂等键以 `ShipmentExpenseId` 为核心，重复确认或同步不能生成重复应付。

## 11. 删除与取消规则

- 草稿费用且未生成应付：允许删除自定义费用。
- 固定费用不物理删除，只允许金额改为 0。
- 已生成应付但无付款：允许取消费用并同步撤销应付。
- 已有付款或抵扣：不允许直接删除费用。
- 金额调低时按超付规则生成预付款。
- 取消出运单时，必须保证所有费用已妥善处理。

## 12. 错误提示

必须提供明确业务错误：

- `海运费金额大于 0，请选择供应商`
- `该费用名称已存在`
- `出运单币种与费用币种不一致`
- `该商品条码已存在`
- `已有付款，不能直接删除该费用`
- `财务同步失败，本次确认已回滚`
- `不同币种的供应商预付款不能自动抵扣`

## 14. 验收标准

至少覆盖：

1. 草稿保存费用，不生成应付。
2. 确认后四条费用分别生成四条应付。
3. 自定义费用正常生成应付。
4. 重复确认不产生重复应付。
5. 修改费用金额后应付同步更新。
6. 修改供应商后正确处理原供应商付款与预付款。
7. 已付款金额高于新费用时生成供应商预付款。
8. 新应付自动抵扣同供应商、同币种预付款。
9. 不同币种预付款不自动抵扣。
10. 同类费用不能重复添加。
11. 金额大于 0 且未选供应商时禁止确认。
12. 商品明细变更后重新汇总立方数、毛重、净重。
13. 人工调整最终值后保留计算值。
14. 商品条码可修改，重复条码被拒绝。
15. 已删除商品条码仍不可复用。
16. 同步中任一步失败时整次事务回滚。
17. Docker 构建通过。
18. 数据库升级通过。
19. API 编译通过。
20. Web 编译通过。

## 15. 非本次范围

本次不包含：

- 多币种费用拆分。
- 同一种费用多行或多个供应商。
- 预付款跨币种折算。
- 自动生成会计总账凭证。
- 第三方支付或银行自动对账。

这些内容可在后续财务增强版本中单独设计。
